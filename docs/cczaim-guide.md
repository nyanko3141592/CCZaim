# Claude Code で簡単 Zaim 管理

家計簿アプリ Zaim のデータを、Claude Code のスラッシュコマンドで操作・分析する。
CCZaim はそのためのツールキットです。

この記事では、セットアップから各コマンドの使い方、活用例まで丁寧に解説します。

## CCZaim とは

CCZaim は、**Zaim API のクライアント + Claude Code のスラッシュコマンド**を1つのリポジトリにまとめたものです。

```
クローン → API キー設定 → 認証 → あとは Claude に話しかけるだけ
```

Zaim アプリは「記録」に強い。でも「分析」は弱い。
そこを Claude Code が補います。

### CCZaim でできること

| やりたいこと | コマンド |
|---|---|
| 今月の家計を一目で把握 | `/dashboard` |
| 「先月のコンビニ代」を調べる | `/zaim-search` |
| 「ランチ1200円」を記録する | `/zaim-add` |
| 未分類の取引を整理する | `/zaim-categorize` |
| 月次・四半期レポートを作る | `/zaim-report` |
| 支出の見直し・貯蓄目標を相談 | `/zaim-fp` |
| カテゴリ別の予算を管理 | `/zaim-budget` |
| 口座一覧を確認 | `/zaim-accounts` |

## セットアップ

### 必要なもの

- Python 3.10 以上
- [Zaim アカウント](https://zaim.net/)
- [Zaim Developer アカウント](https://dev.zaim.net/)（API キー取得用）
- [Claude Code](https://claude.ai/code)

### 手順

#### 1. リポジトリをクローン

```bash
git clone https://github.com/nyanko3141592/CCZaim.git
cd CCZaim
```

#### 2. 依存パッケージをインストール

```bash
pip3 install -r requirements.txt
```

内容は `requests-oauthlib`（OAuth認証）と `python-dotenv`（環境変数管理）の2つだけです。

#### 3. Zaim Developer Center でアプリ登録

1. [dev.zaim.net](https://dev.zaim.net/) にアクセス
2. Zaim アカウントでログイン
3. 「新しいアプリケーションを登録」から登録
4. アプリ名は何でもOK（例: `CCZaim`）
5. コールバック URL は `http://localhost`
6. 登録後、**Consumer Key** と **Consumer Secret** をメモ

#### 4. 環境変数を設定

```bash
cp .env.example .env
```

`.env` を開いて、先ほどの Key / Secret を貼り付けます:

```
ZAIM_CONSUMER_KEY=ここにConsumerKey
ZAIM_CONSUMER_SECRET=ここにConsumerSecret
```

#### 5. OAuth 認証を実行

```bash
python3 zaim_client.py auth
```

ブラウザが自動で開きます。Zaim にログインして「許可」を押すと、`http://localhost?...` にリダイレクトされます。
ブラウザのアドレスバーに表示された URL 全体をコピーして、ターミナルに貼り付けてください。

成功すると `tokens.json` が保存されます（`.gitignore` に含まれているので、誤って公開される心配はありません）。

#### 6. 動作確認

```bash
python3 zaim_client.py user     # ユーザー情報
python3 zaim_client.py money    # 最近の取引
```

データが表示されたらセットアップ完了です。

## 各コマンドの詳細

### `/dashboard` — 今月の家計ダッシュボード

```
> /dashboard
```

今月1日から今日までの全取引を集計して、以下を表示します:

- **収支サマリー**: 収入・支出・差引
- **カテゴリ別支出**: 棒グラフ付きランキング
- **高額支出 TOP5**: 大きな出費を一目で確認
- **日別推移**: どの日にいくら使ったか
- **未分類件数**: 分類が必要な取引の数

表示後は、Claude が次のアクションを提案します:

```
ダッシュボードを表示しました。次に何をしますか？
○ 特定カテゴリを深堀り
○ 先月と比較する
○ 未分類を整理する
○ 終了
```

「特定カテゴリを深堀り」を選ぶと、さらにどのカテゴリを見たいか選択肢が出ます。
キーボードで選ぶだけでどんどん深堀りできます。

---

### `/zaim-search` — 取引データ検索

自然言語で取引を検索できます。

```
> /zaim-search
> 先月のコンビニ支出
```

他にもこんな聞き方ができます:

- 「3月の食費を調べて」
- 「今月1万円以上の支出」
- 「直近の Amazon 購入」
- 「去年の交際費」

Claude が日付範囲・カテゴリ・金額条件を解釈して、Zaim API に問い合わせます。
結果は表形式で表示され、合計金額と件数のサマリーが付きます。

```
| 日付 | カテゴリ | ジャンル | 金額 | 場所 | メモ |
|---|---|---|---|---|---|
| 2024-03-05 | 食費 | コンビニ | ¥580 | セブンイレブン | |
| 2024-03-12 | 食費 | コンビニ | ¥1,200 | ファミマ | お弁当 |
| ... | ... | ... | ... | ... | ... |

合計: ¥4,280（8件）
```

---

### `/zaim-add` — 支出・収入の登録

自然言語で支出を記録できます。

```
> /zaim-add
> 今日ランチで1200円使った
```

Claude が入力を解析して、登録内容を確認表示します:

```
以下の内容で登録します:

| 項目 | 内容 |
|---|---|
| 種別 | 支出 |
| 金額 | ¥1,200 |
| 日付 | 2024-04-04 |
| カテゴリ | 食費 |
| ジャンル | 外食 |

よろしいですか？
```

「はい」で登録完了。**必ず確認画面が出るので、誤登録の心配はありません。**

こんな言い方にも対応:
- 「スタバでコーヒー550円」
- 「3/15に本を2000円で買った」
- 「給料30万円入った」（収入も可）

---

### `/zaim-categorize` — 未分類取引の対話的分類

Zaim に取り込まれた銀行・クレカの明細は、カテゴリが「未分類」になっていることがあります。
このコマンドで対話的に分類できます。

```
> /zaim-categorize
```

#### 進め方の選択

未分類が5件以上ある場合、まず進め方を聞かれます:

```
未分類が12件あります。どう進めますか？
○ 1件ずつ確認（Recommended）
○ AI提案を一括表示
○ 高額順に上位5件だけ
```

#### 1件ずつの分類フロー

各取引について、店名と金額からAIがカテゴリを推定し、選択肢を提示します:

```
「セブンイレブン」¥580（2024-03-05）のカテゴリは？
○ 食費（Recommended） — 店名「セブンイレブン」から推定
○ 日用品 — コンビニで日用品を買った場合
○ スキップ — あとで分類する
```

カテゴリを選ぶと、次はジャンル:

```
食費 のジャンルは？
○ コンビニ（Recommended）
○ 食料品
○ その他食費
```

選択するとすぐに Zaim が更新され、次の取引に進みます。

### `/zaim-report` — 期間レポート生成

詳細な家計レポートを生成します。

```
> /zaim-report
```

#### 期間の選択

```
どの期間のレポートを作りますか？
○ 先月（Recommended）
○ 今月（途中経過）
○ 直近3ヶ月
○ 今年
```

#### 観点の選択（複数選択可）

```
どの観点を重視しますか？
☐ カテゴリ別分析（Recommended）
☐ 前期比較
☐ 高額支出の洗い出し
☐ 日別パターン分析
```

選んだ内容に応じて、マークダウン形式のレポートが生成されます:

- **総括**: 収入・支出・差引・件数
- **カテゴリ別支出**: 割合付きの棒グラフ
- **前期比較**: 各カテゴリの増減と増減率
- **高額支出**: 個別の大きな出費一覧
- **日別パターン**: 曜日別平均のグラフ
- **AIの所感**: データから読み取れるトレンドや改善ポイント

レポート表示後も、さらに深堀りやFP相談への導線が用意されています。

### `/zaim-fp` — FP 相談

ファイナンシャルプランナー的な視点で、家計データに基づいたアドバイスをもらえます。

```
> /zaim-fp
```

#### 4つの相談メニュー

```
どんな相談をしますか？
○ 支出の見直し（Recommended）
○ 貯蓄・投資の目標設定
○ 固定費チェック
○ ライフイベント試算
```

#### A: 支出の見直し

実データから削減余地のあるカテゴリ TOP3 を特定。各カテゴリのジャンル別内訳を表示し、具体的な削減目標を対話的に設定できます。

```
食費が¥38,200（全体の32%）です。内訳を見ますか？
○ ジャンル別の内訳を見る（Recommended）
○ 日別の推移を見る
○ 次のカテゴリへ
```

```
食費の目標を設定しますか？
○ 先月比10%削減（¥34,400/月）（Recommended）
○ 具体的な金額を設定
○ 目標は設定しない
```

#### B: 貯蓄・投資の目標設定

直近3ヶ月の収支データから貯蓄余力を自動計算。目標金額と期間を対話的に設定して、達成までのシミュレーションを表示します。

#### C: 固定費チェック

毎月繰り返し発生している支出を自動検出。通信費・サブスク・保険料などを一覧表示し、見直し対象を複数選択できます。

#### D: ライフイベント試算

引っ越し・転職・結婚などの大きな出費について、現在の収支データに基づいた試算を行います。

> **注意**: FP 相談はあくまで一般的な情報提供です。具体的な金融商品の推奨は行いません。

### `/zaim-budget` — カテゴリ別予算管理

Zaim API には予算機能がないため、CCZaim 独自にローカルで管理します。

```
> /zaim-budget
```

#### 予算消化チェック

設定済みの予算に対する今月の進捗を表示:

```
## 今月の予算消化状況（4/15時点）

| カテゴリ | 予算 | 実績 | 残り | 消化率 |
|---|---|---|---|---|
| 食費 | ¥40,000 | ¥28,500 | ¥11,500 | 71% ⚠️ ペース超過 |
| 日用品 | ¥10,000 | ¥3,200 | ¥6,800 | 32% ✅ 順調 |
| 趣味 | ¥15,000 | ¥15,800 | -¥800 | 105% 🔴 超過！ |
```

超過したカテゴリがあれば対応を聞かれます:

```
趣味が¥800超過しています。どうしますか？
○ 内訳を確認する（Recommended）
○ 今月は許容する
○ 予算を増額する
```

#### 予算の設定

過去の実績ベース・収入からの逆算・手入力の3方式から選べます。
実績ベースなら、直近3ヶ月の平均を元にカテゴリごとに提案してくれます。

## CLI 単体でも使える

Claude Code を使わなくても、`zaim_client.py` はコマンドラインツールとして動きます。

```bash
# 最近の取引を表示
python3 zaim_client.py money

# 期間指定で全件取得
python3 zaim_client.py money --start 2024-01-01 --end 2024-03-31 --all

# 月別サマリー
python3 zaim_client.py summary --start 2024-01-01

# 今月のダッシュボード（JSON出力）
python3 zaim_client.py dashboard

# 未分類の取引
python3 zaim_client.py uncategorized

# カテゴリ / ジャンル / 口座マスタ
python3 zaim_client.py categories
python3 zaim_client.py genres
python3 zaim_client.py accounts
```

スクリプトから呼ぶこともできます:

```python
import zaim_client

s = zaim_client.get_session()
items = zaim_client.get_all_money(s, "2024-03-01", "2024-03-31")

summary = zaim_client.monthly_summary(items)
breakdown = zaim_client.category_breakdown(items)
```

## よくある質問

### Q. Zaim の有料プラン（プレミアム）は必要？

不要です。API は無料プランでも利用可能です。

### Q. API キーが漏れたらどうなる？

あなたの家計簿データに第三者がアクセスできてしまいます。
`.env` と `tokens.json` は `.gitignore` に含まれているので、通常の git 操作では漏れませんが、取り扱いには注意してください。

### Q. 家族の Zaim アカウントのデータも見られる？

API は認証したアカウントのデータのみ扱えます。家族のデータを見るには、家族のアカウントで別途認証が必要です。

### Q. データの登録・更新で Zaim アプリ側にも反映される？

はい。API で行った操作は即座に Zaim アプリに反映されます。

### Q. 自分でスラッシュコマンドを追加できる？

もちろんです。`.claude/commands/` に新しいマークダウンファイルを置くだけです。
`zaim_client.py` の関数を使えば、API 呼び出しのコードを書く必要はほとんどありません。

## まとめ

CCZaim のコンセプトは単純です:

- **記録は Zaim アプリに任せる** — レシート撮影、銀行自動連携、手入力。Zaim が得意なこと
- **分析は Claude Code に任せる** — 集計、可視化、レポート、アドバイス。AI が得意なこと

この2つを API で繋ぐだけで、家計管理の「めんどくさい」がかなり減ります。
